## Purpose

The `brain/` folder is ARL's distilled operating intelligence.

It is not a report folder.  
It is not a dumping ground.  
It is not a second office ledger.  
It is not a place for random doctrine sprawl.

The Brain Guidebook system exists so every future run gets smarter without burying useful lessons inside old reports.

---

## Brain Folder Law

```text
Reports record what happened.
Office records official decisions and ledgers.
Roadmap records how to build.
Blueprint records what the final product must become.
Brain records durable lessons that improve future thinking.
```

A lesson enters the brain only if it improves future coding, research, prompt-building, output design, risk handling, or failure prevention.

## Brain Update Rule

Every serious run must ask:

```text
Did this run discover a durable lesson?
Which guidebook should absorb it?
Is it new, or already covered?
Does it prevent a known failure?
Does it create a concrete rule, test, or falsifier?
```

If the answer is weak, do not add it.

---

## Anti-Sprawl Rule

Do not create new brain files casually.

Only create a new guidebook if:
- the topic is durable,
- it cannot cleanly fit an existing guidebook,
- it will be read by future runs,
- it prevents real drift or failure.

Otherwise update an existing guidebook.

---

## Source Authority

The Brain Guidebook system does not outrank active source or proof.

Authority order:

```text
1. Active Work Area source files
2. Compile/runtime/output evidence
3. Office decisions and ledgers
4. Roadmap build law
5. Blueprint final-product law
6. Brain distilled lessons
7. Git Archives inheritance evidence
8. Old chats / memory / summaries
```

If Brain conflicts with active source or proof, log a contradiction and resolve it.
