# ARL Roadmap Completion Status Law

Run introduced: ARL-RUN010G  
Status: Mandatory office rule  
Linked file: Work Area/roadmap/ARL_ROADMAP_COMPLETION_STATUS.md

---

## Law

Every serious ARL run must read and update:

```text
Work Area/roadmap/ARL_ROADMAP_COMPLETION_STATUS.md
```

This file tracks how complete each phase/module is and how complete the whole roadmap is.

---

## Why This Exists

ARL must not confuse:

```text
planned
scaffolded
compiled
runtime tested
accepted
locked
complete
```

They are different evidence states.

---

## Update Requirement

A run must update roadmap completion status if it changes:

```text
source code
roadmap
blueprint
office records
brain guides
runtime IO
account/exposure
universe
layers
outputs
testing
Git/prompt/agent workflow
```

If no percentage changes, the run must still add a ledger note explaining why.

---

## Prompt Requirement

Every serious prompt must include:

```text
Read and update Work Area/roadmap/ARL_ROADMAP_COMPLETION_STATUS.md.
Do not skip completion tracking.
```

---

## Failure Condition

A serious run that changes ARL but does not update or explicitly justify no update to roadmap completion status is incomplete.
