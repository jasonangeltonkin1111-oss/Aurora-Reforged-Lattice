# ARL Office Pack — RUN001 Foundation

## Purpose

This `office/` folder is the record-control layer for **Aurora Reforged Lattice**.

It exists so every roadmap decision, change, risk, test, research finding, migration candidate, lock, and build step is readable before future work begins.

This folder is not runtime output.
This folder is not old ASC authority.
This folder is not proof that ARL compiles, runs, trades, or has edge.

It is the planning and governance layer.

## Mandatory rule

Before any serious ARL work, the active chat / Codex / agent must read the relevant office files first:

1. `ARL_OFFICE_INDEX.md`
2. `ARL_DECISIONS.md`
3. `ARL_ROADMAP.md`
4. `ARL_CHANGE_LEDGER.md`
5. `ARL_RISK_LEDGER.md`
6. `ARL_RESEARCH_LEDGER.md`
7. `ARL_MIGRATION_REGISTER.md`
8. `ARL_ACCEPTANCE_TESTS.md`
9. `ARL_LOCK_POLICY.md`

Any meaningful change must update:
- `ARL_CHANGE_LEDGER.md`
- `ARL_DECISIONS.md` if a design decision changed
- `ARL_RISK_LEDGER.md` if risk changed
- `ARL_ACCEPTANCE_TESTS.md` if behavior changed
- the relevant lock file once the module is proven and locked

## Office authority

Office files can guide work, but active source files and compile/runtime evidence outrank office prose.

Authority order:

1. Current user instruction
2. Active ARL source files
3. Compile/static validation
4. Runtime output generated by current ARL source
5. Office files
6. Archives and old ASC outputs
7. AI inference / memory

## Public safety

Do not commit real private runtime outputs with live account number, account name, server paths, terminal paths, trade history, broker account details, or personal machine paths.

Samples must mask sensitive values.
