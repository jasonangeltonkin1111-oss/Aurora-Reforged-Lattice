## Primary Root-Cause Answers

1. Market Board Top10 text is generated in `mt5/artifacts/ASC_MarketBoardWriter.mqh` through `ASC_BuildMarketBoardArtifact()`, `ASC_BoardBuildAllMarketsTopRows()`, `ASC_BoardAppendCommandBlock()`, and symbol block helpers.
2. MarketBoardWriter consumes `full_rows[]`; it filters visible rows but does not read generated folders.
3. A durable committed snapshot id is not directly exposed to MarketBoardWriter; RUN231 marks it unknown rather than inventing it.
4. RUN230 duplicate route context is not directly exposed to MarketBoardWriter; RUN231 marks duplicate/parallel fields unknown rather than importing filesystem truth or touching Dispatcher.
5. Source-owned numeric values available: rank, symbol/canonical symbol, classification asset_class/bucket, shortlist/L3/context scores, filter execution score, L2 spread/tick/freshness/liquidity when lawful, correlation peer summary when source-ready.
6. Absent values marked unknown/source-pending: snapshot id, duplicate route instance, parallel route family, section coherence, crypto health, some correlation/exposure when not ready.
7. Top-1 bias was generated in `ASC_BoardAppendCommandBlock()` and related reason helpers.
8. Top-1 wording was downgraded without changing rank/order.
9. SIAM DOSSIER REQUEST BLOCK was added without broad board redesign.
10. TOP10 NUMERIC MATRIX was added from existing values only.
11. No field required L3/L4 formula change; unavailable fields stay unknown or defer to RUN232-RUN235.
12. Smallest safe patch: add two display-only blocks and downgrade output language in MarketBoardWriter; update version/control/report only.
