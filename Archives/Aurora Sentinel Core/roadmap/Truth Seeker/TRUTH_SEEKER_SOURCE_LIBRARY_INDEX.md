# Truth Seeker Source Library Index

**Created:** 2026-04-26  
**Purpose:** Canonical source-library intake map for book, paper, practitioner, and Aurora-internal research material.

## Authority Boundary

The source library is **source material**, not proof. A book title, screenshot, public summary, paper abstract, or prior extraction note can justify investigation. It cannot create a trusted Aurora strategy label by itself.

Future workers must separate four evidence levels:

| Evidence type | Allowed use | Forbidden use |
|---|---|---|
| Screenshot-only title | Bibliographic category, likely source family, search target | Claiming the book was read or extracted |
| Uploaded book/file | Concept extraction, source summary, candidate method card | Long copyrighted text dump or automatic edge proof |
| Public internet/academic source | Research cross-check and test-design support | Live proof or strategy trust |
| Aurora EA zip/source/log evidence | Implementation status, file/function/card/log proof | Guessing implemented/missing without inspection |

## Operator Library Families

The operator-provided screenshots/title list and Aurora reference zip point to these source families.

### 1. Market Efficiency / Randomness / Anti-Overconfidence

Examples: `A Random Walk Down Wall Street`, efficient-market/random-walk material, `Evidence-Based Technical Analysis`, data-snooping/reality-check sources.

Aurora use:
- prevent fake edge and pattern worship
- require objective rules and out-of-sample evidence
- force uncertainty language when evidence is incomplete
- reject labels that cannot survive statistical review

### 2. Adaptive Markets / Regime Change

Examples: `Adaptive Markets`, behavioral finance, `Misbehaving`, `Thinking, Fast and Slow`, `The Hour Between Dog and Wolf`.

Aurora use:
- regime-specific method admission
- strategy-decay and drift review
- model competition under changing environments
- overconfidence / stress / emotional decision guardrails

### 3. Market Microstructure / Auction / Execution Reality

Examples: `Trading and Exchanges`, `Market Microstructure Theory`, `Auction Theory`, `James Dalton Markets in Profile`, `Mind Over Markets`, `Flash Boys`.

Aurora use:
- spread, friction, tick age, quote freshness
- acceptance/rejection and value-vs-price candidates
- session liquidity and execution-quality warnings
- no fake order-flow claims when MT5 only supplies proxy data

### 4. Systematic / Quant / Algo Strategy Design

Examples: `Quantitative Trading`, `Algorithmic Trading`, `Quantitative Trading Strategies Using Python`, `Systematic Trading`, `Statistical Arbitrage`, `Mathematics of Money Management`, walk-forward/optimization material.

Aurora use:
- exact rule definition
- feature/outcome logging
- backtest and walk-forward validation design
- anti-overfit safeguards and reject conditions

### 5. Financial Machine Learning / AI

Examples: `Advances in Financial Machine Learning` and later AI/ML research.

Aurora use:
- feature dataset spine
- outcome labels
- purging/embargo concepts where relevant
- meta-labeling and feature-importance research candidates
- AI Forex GPT governance, advisory-only until proven

### 6. Technical Analysis / Price Action / Chart Structure

Examples: Adam Grimes, John Murphy, Bulkowski, Steve Nison, Elder, classical technical-analysis texts, indicator-settings material.

Aurora use:
- structure labels: trend, pullback, breakout, failed break, reclaim, rotation, compression, expansion, exhaustion
- candlestick/pattern evidence only as weak or candidate support unless validated
- precise formula/rule definition before any operator-facing label survives

### 7. Volatility / Options / Risk / Tail Risk

Examples: `Option Volatility and Pricing`, `Dynamic Hedging`, `Volatility Trading`, `The Black Swan`, `Against the Gods`.

Aurora use:
- volatility regime, volatility expansion/compression
- convexity, ruin, gap, disorder, and tail-risk warnings
- spread-to-room / geometry risk
- no ruin paths and prop-firm caution

### 8. Fundamental / Macro / Expected Returns

Examples: `Expected Returns`, `How Countries Go Broke`, `Alchemy of Finance`, `Common Stocks and Uncommon Profits`, `The Intelligent Investor`, `One Up On Wall Street`, `The Little Book That Beats the Market`.

Aurora use:
- asset-class and macro warning context
- longer-horizon caution
- not a primary scalp entry signal
- possible swing/macro risk annotation only after horizon gating

### 9. Forecasting / Decision Science / Behavior

Examples: `Superforecasting`, `Thinking in Bets`, `Thinking, Fast and Slow`, behavioral decision science.

Aurora use:
- probability language
- confidence calibration
- decision journal and forward-review structure
- avoiding certainty language
- documenting alternative hypotheses

## Intake Rules

1. If the book/file is uploaded, inspect it conceptually and extract method ideas without reproducing long copyrighted text.
2. If only screenshots or titles exist, classify source family and uncertainty only.
3. If the Aurora EA zip contains prior extraction ledgers/cards, treat them as implementation-reference evidence, not proof that every concept is complete.
4. Every source concept must go through the Book-to-Aurora Crosswalk before a method card or future implementation proposal.
5. Every useful concept must be classified as implemented, partial, misplaced, label-only, duplicate/conflicting, missing card, missing from Aurora, rejected, or future-test candidate.
6. No future worker may claim total extraction unless uploaded source files and Aurora file/card/log proof support it.

## Required Source Intake Output

Every source-library run must report:

- source files/titles actually inspected
- screenshot-only titles treated as uncertain
- uploaded books/files treated copyright-safely
- public research used
- Aurora EA zip/source areas inspected
- concepts extracted
- concepts rejected
- crosswalk rows updated
- method ledger rows updated
- strategy cards created/updated/rejected
- no-full-auto and no-execution-creep declaration
