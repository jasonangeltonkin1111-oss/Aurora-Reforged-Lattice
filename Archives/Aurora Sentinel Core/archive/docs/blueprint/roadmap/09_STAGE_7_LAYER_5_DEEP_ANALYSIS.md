# ASC Stage 7 — Layer 5 Deep Analysis

## Purpose

This document defines Stage 7 of the Aurora Sentinel Scanner (ASC) rebuild roadmap.

Stage 7 is the Layer 5 deep-analysis stage.
Its job is to turn lawful promoted membership into deeper rolling truth for the selected set only.

If Stage 6 answers which survivors deserve promotion, Stage 7 answers the next lawful question:

> for the symbols that have earned privileged attention, what deeper rolling truth should ASC maintain now?

This stage exists because deeper analysis has value only after scope is already constrained by real shortlist authority.
Without that boundary, “deep analysis” becomes a second scanner hiding behind richer language.

---

## Stage objective

Stage 7 establishes lawful Layer 5 runtime behavior inside the governed runtime spine.

It defines and/or implements:
- executable Layer 5 deep state for promoted symbols
- promoted-set-only deep service
- rank-sensitive deep-service cadence within the promoted set
- deep freshness posture
- deep continuity/last-good posture where assigned
- lawful Layer 5 committed artifact publication
- dirty-routing outputs from Layer 5 into later structural consumer eligibility without granting upstream authority back into selection or filtering

The goal is not to widen the scanner.
The goal is to make depth real only where the architecture has already earned the right to spend it.

---

## Why Stage 7 matters

Without Stage 7, the selected set exists but deeper promoted truth does not.
That creates two recurring bad shapes:
- rich views and downstream interpretation begin improvising deep reads ad hoc
- selection logic starts absorbing deeper context because no explicit deep owner exists

Stage 7 matters because it makes deeper truth a real owned layer instead of a loose ambition.

If Stage 7 is weak:
- promoted-set scope widens unpredictably
- deep cadence becomes noisy or UI-driven
- persistence pressure rises without controlled benefit
- selected-set consumers cannot distinguish structural depth from tactical improvisation
- Layer 4 authority becomes vulnerable to back-pressure from deeper logic

## Admission doctrine in Stage 7

### Layer 5 depends on Layer 4 promoted membership
Stage 7 must make it real that Layer 5 is not a discovery surface.
It is admitted based on already committed promoted membership from Layer 4.

### Promotion loss removes deep entitlement
If a symbol leaves the promoted set, Stage 7 must preserve the rule that it loses active deep-service entitlement.
Previously committed deep truth may remain visible in controlled continuity-safe ways where architecture permits, but active service rights are gone.

### Recovery exceptions must stay explicit and narrow
If later recovery logic needs to touch recently demoted or transitioning members, that must remain explicit and bounded.
Stage 7 must not quietly normalize such behavior as normal deep scope.

### Strict decision order must be executable
Stage 7 must implement entitlement evaluation in strict gate order:

1. symbol is in Layer 4 promoted set (**hard gate**)
2. service mode is explicit user-triggered Deep Analysis for that selected symbol
3. rank-sensitive cadence applies only to already admitted symbols and never widens scope
4. demotion removes active entitlement immediately; only bounded continuity view may remain

Any failing earlier gate must terminate entitlement.
No rank, UI focus, or urgency signal may bypass gate 1.

## Promoted-set-only doctrine in Stage 7

### The central boundary must become executable now
Stage 7 must structurally enforce that Layer 5 operates only on the current promoted set.

### Why this matters
Layer 5 is allowed to be more expensive only because the selected set is bounded first.
If that boundary fails, the whole scanner starts expanding expensive work without lawful authority.

### The selected set is the price of depth
Stage 7 must preserve the discipline that deeper truth is earned by selection, not by curiosity, focus, or local convenience.

## Deep-scope doctrine in Stage 7

### Deep scope must remain intentional
Stage 7 should implement only the deeper fields/surfaces that truly belong to promoted-set analysis.
These may include architecture-assigned deeper packets, rolling context, or promoted-only analytical fields.

### Deep scope must not become a dumping ground
If Stage 7 starts absorbing every field that looks “useful but richer,” it has already drifted.
Fields that belong naturally to earlier layers must stay there.

### Deep scope should improve promoted understanding materially
The stage should create depth that later consumers can actually use, not prestige-data with weak structural value.

---

## Rank-sensitive cadence doctrine in Stage 7

### Rank-sensitive cadence must become real now
Stage 7 should make it executable that different promoted members may receive different deep-service cadence inside the promoted set.

### Rank-sensitive cadence may influence
- deep refresh frequency
- subpacket priority where architecture defines multiple deep surfaces
- which promoted members are serviced first under bounded pressure

### Rank-sensitive cadence may not influence
- who is promoted
- rights for non-promoted symbols
- write-at-display-speed behavior
- hidden shortlist redefinition

### Scope stays fixed even when cadence differs
This is the key law:
**cadence may vary inside the promoted set; scope does not widen because of that.**

## Freshness doctrine in Stage 7

### Deep freshness must become real now
The runtime should be able to answer:
- when a promoted symbol’s deep truth was last refreshed
- whether it is fresh, aging, stale, degraded, or continuity-backed
- whether different deep subcomponents carry different freshness where architecture allows that distinction

### Freshness does not imply constant service
Stage 7 should preserve the rule that deep freshness is maintained by selective waves, not by pulse-hungry obsession.

### Deep freshness must remain inspectable
Later operator surfaces and summaries will depend on this.
It must become structural now, not improvised later.

## Continuity doctrine in Stage 7

### Why continuity matters here
Deep truth is more expensive than earlier layers and therefore more vulnerable to controlled aging between service waves.
Stage 7 should make continuity/last-good posture explicit where assigned.

### Continuity must remain honest
A continuity-backed deep artifact is not freshly refreshed deep truth.
Stage 7 must preserve that distinction strongly.

### Controlled continuity only
Continuity should apply only to fields and cases the architecture explicitly allows.
It must not become a blanket excuse to preserve old deep state indefinitely.

---

## Scheduling doctrine in Stage 7

### Layer 5 should be promoted-wave driven
Stage 7 should make deep service structurally wave-based across the admitted promoted set rather than everywhere, every beat.

### Service should respect rank, backlog, and governor posture
Deep service should be scheduled based on:
- current promoted membership
- rank/service-tier posture
- stale/degraded deep pressure inside the promoted set
- available credits and mode posture under the governor

### Deep service should not be UI-led
Opening a page or focusing a symbol must not be enough to begin deep service for a symbol that is not already promoted.

### Calmness through narrowness
A healthy Layer 5 may be active, but its activity should still feel bounded because the selected set is bounded.

## Focus doctrine in Stage 7

If a symbol is already promoted and already entitled to deep service, later stages may allow focus to influence how already-owned deep truth is consumed or tactically refreshed in bounded ways.

## Entitlement state machine requirements in Stage 7

Stage 7 must implement explicit entitlement states with explicit transitions, dirty triggers, and write triggers.

### Required states
- `NOT_ENTITLED`
- `ADMITTED_PENDING_TRIGGER`
- `ACTIVE_ENTITLED`
- `ACTIVE_RANK_SCHEDULED`
- `DEMOTED_CONTINUITY_ONLY`
- `EXPIRED_NOT_ENTITLED`

### Required transition rules
- `NOT_ENTITLED -> ADMITTED_PENDING_TRIGGER` only on confirmed Layer 4 promotion
- `ADMITTED_PENDING_TRIGGER -> ACTIVE_ENTITLED` only on explicit Deep Analysis trigger
- `ACTIVE_ENTITLED <-> ACTIVE_RANK_SCHEDULED` only as cadence scheduling changes within admitted scope
- `ADMITTED_PENDING_TRIGGER|ACTIVE_ENTITLED|ACTIVE_RANK_SCHEDULED -> DEMOTED_CONTINUITY_ONLY` on promotion loss
- `DEMOTED_CONTINUITY_ONLY -> EXPIRED_NOT_ENTITLED` on continuity expiry or continuity-policy close
- any return to active states requires fresh re-promotion first

### Required dirty triggers by state
- `NOT_ENTITLED`: dirty entitlement-change consumers; clear active queue tokens
- `ADMITTED_PENDING_TRIGGER`: dirty on-demand eligibility routing only; no auto deep-build dirty
- `ACTIVE_ENTITLED`: dirty Layer 5 deep build queue and downstream consumer eligibility-pending
- `ACTIVE_RANK_SCHEDULED`: dirty scheduler ordering/next-due metadata only inside admitted set
- `DEMOTED_CONTINUITY_ONLY`: dirty continuity/degraded surfaces and cancel active deep jobs
- `EXPIRED_NOT_ENTITLED`: dirty continuity-clear surfaces and final entitlement posture

### Required write triggers by state
- `NOT_ENTITLED`: no deep-build write; optional marker only
- `ADMITTED_PENDING_TRIGGER`: optional entitlement metadata write (`admitted=true`, `active=false`)
- `ACTIVE_ENTITLED`: structural deep write only after admitted deep work produces material change
- `ACTIVE_RANK_SCHEDULED`: cadence metadata writes allowed; scope/admission writes forbidden
- `DEMOTED_CONTINUITY_ONLY`: bounded continuity/degraded marker write allowed; new deep rebuild write forbidden
- `EXPIRED_NOT_ENTITLED`: optional continuity-expiry marker; no deep payload write

## What Stage 7 makes newly admissible
## Stage 7 separation law

Stage 7 must preserve the global-vs-on-demand split explicitly:
- Layers 1 through 4 continue as the globally governed broad engine
- Deep Analysis remains isolated from that broad engine

Stage 7 may deepen selected symbols.
It may not convert Deep Analysis into an automatic all-symbol heartbeat service.

### HUD-triggered activation law
Where Deep Analysis is exposed operationally, it must be triggered explicitly by user action and applied only to the selected symbol.
Focus or HUD interaction may request depth; it may not authorize global deep churn.


Once Stage 7 is complete, the following become admissible:
- Stage 8 persistence/artifact-family hardening against real Layer 1 through Layer 5 truth
- later promoted deep structural views consuming committed Layer 5 truth
- later promoted composition and summary work consuming real promoted depth

What does **not** become automatically admissible:
- widening deep scope because the infrastructure now exists
- letting deep outputs back-pressure shortlist authority
- treating focus as proof that deep scope should be broader

## Common failure patterns Stage 7 must prevent

### Pattern 1 — Universe creep
Deep service starts touching symbols outside the promoted set.

Result:
- runtime cost rises sharply
- earlier-layer boundaries lose meaning
- deep scope becomes lawless

### Pattern 2 — Authority leak backward
Deep outputs begin influencing selection implicitly.

Result:
- shortlist authority weakens
- causality becomes harder to inspect

### Pattern 3 — Focus entitlement bug
A non-promoted symbol gains deep service because the operator is looking at it.

Result:
- UI attention becomes runtime authority
- scope widens unpredictably

### Pattern 4 — Same-speed deep writes
Deep publication begins to follow tactical or display-driven cadence too closely.

Result:
- persistence pressure rises
- calm publication is lost

### Pattern 5 — Prestige-data sprawl
Stage 7 keeps absorbing expensive deep surfaces because they look advanced, not because they serve the promoted set meaningfully.

Result:
- Layer 5 becomes a dumping ground
- depth loses purpose
- costs rise without structural leverage

## Stage 7 invalid completions

Stage 7 is **not** complete if any of the following are true:
- Layer 5 deep truth is still mostly ad hoc or view-driven
- promoted-only scope is weak or easily bypassed
- rank-sensitive cadence widens scope or becomes ambiguous
- deep freshness/continuity posture is mostly implicit
- focus still creates deep entitlement for non-promoted symbols
- committed Layer 5 truth does not land lawfully
- Layer 5 has already begun influencing selection authority
- later consumer stages would still need to improvise deep truth themselves
