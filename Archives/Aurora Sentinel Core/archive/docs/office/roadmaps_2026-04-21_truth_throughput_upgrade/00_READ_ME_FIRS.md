# ASC Truth + Throughput Upgrade — Read Me First

## Purpose

This folder is the active roadmap family for the next upgrade cycle.

It exists because the product now needs a stricter and more practical roadmap than the last cycle:

- ordinary dossier behavior is not yet lawful at scale
- solved deep paths must be protected from regression
- throughput and publication efficiency must improve
- Layers 3–6 need better truth quality without drifting into gates
- cleanup must stop causing rework loops

## What this roadmap family is for

This roadmap family is for:

- restoring lawful ordinary dossier family behavior
- protecting solved deep and Global Top 10 paths
- improving write-lane throughput and publication efficiency
- polishing Layer 0–2 truth where it sharpens downstream accuracy
- upgrading Layer 3–6 ranking/truth quality using architecture-safe ideas
- aligning operator surfaces to the upgraded model
- validating in batches with honest proof classes

## What this roadmap family is not

This roadmap family is not:

- a broad redesign program
- a HUD redesign program
- a second analysis-engine plan
- a second owner-path plan
- a gate-heavy scoring roadmap
- a “test every commit” panic workflow
- a cosmetic wording-only initiative

## Core upgrade law

The system remains ranking-led, not gate-led.

Use:

- score
- lift
- drag
- influence
- ordering
- shortlist quality
- contradiction surfacing
- support truth

Do not organize the upgrade around binary allow/block gates unless a lane is explicitly about lawful publication admission.
