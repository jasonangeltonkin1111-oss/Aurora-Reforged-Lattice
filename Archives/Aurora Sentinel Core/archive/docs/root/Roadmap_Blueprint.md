DOSSIER SYSTEM RECOVERY PROGRAM
AURORA SENTINEL SCANNER
DETAILED PRODUCER-FIRST ROADMAP

======================================================================
0. CORE THESIS
======================================================================

The dossier program is NOT a "text output" project.
It is a PRODUCER -> ADAPTER -> COMPOSER -> PUBLICATION -> VALIDATION project.

The dossier must never become a truth owner.

Every dossier field must answer ALL of these:

1) Which engine/module owns this truth?
2) Which contract/packet carries this truth?
3) Which dossier adapter formats this truth?
4) Which publication path lands this truth?
5) Which validation path proves this truth?

If any field cannot answer those five questions, that field is architecturally unsafe.

======================================================================
1. NON-NEGOTIABLE LAWS
======================================================================

1. Dossiers are CONSUMERS.
   - They may format truth.
   - They may not invent truth.
   - They may not re-derive truth casually.
   - They may not outrank compile-reachable runtime ownership.

2. Layer boundaries must remain explicit.
   - L0 = identity/classification/metadata foundation
   - L1 = market state / timing / confidence / wake model
   - L2 = open symbol snapshot and broker-spec packet family
   - L3 = candidate filtering
   - L4 = shortlist selection
   - L5 = deep selective analysis

3. Layer 5 remains:
   - explicit-trigger only
   - selected / entitled only
   - not widened into background universe service
   - allowed to preserve prior deep truth lawfully
   - forbidden from becoming auto-hydrated across the whole universe

4. Closed-market symbol dossier law:
   - opening frame visible
   - L0 visible
   - L1 visible
   - L2 visible
   - stop cleanly
   - no L3/L4/L5 headings
   - no placeholder blocks for omitted later layers

5. Open-market symbol dossier law:
   - opening frame visible
   - L0 visible
   - L1 visible
   - L2 visible
   - L3 visible
   - L4 visible
   - L5 scaffold visible
   - L5 remains explicit deep-analysis domain

6. Startup / warmup dossier law:
   - near-instant full-universe coverage for Layer 0-2
   - cheap lawful dossier first
   - rich continuation later
   - shell/existence path is bootstrap fallback, NOT final visible product

7. Codex pass law:
   - one pass type only
   - audit OR design OR implementation OR review
   - never mixed casually
   - never "fix everything"
   - every pass ends with a stop gate

======================================================================
2. MASTER SYSTEM MODEL
======================================================================

The dossier system is composed of FIVE STRANDS.

----------------------------------------------------------------------
STRAND A — PRODUCER ENGINES
----------------------------------------------------------------------

These own truth.

A1. L0 Producer Family
    Owns:
    - symbol identity
    - broker symbol
    - canonical symbol
    - display name
    - classification
    - broker descriptors
    - metadata normalization

A2. L1 Producer Family
    Owns:
    - market state
    - state reason
    - confidence
    - next due
    - last checked
    - bid/ask presence
    - wake/recheck posture

A3. L2 Producer Family
    Owns:
    - quote snapshot
    - day structure
    - ATR / friction / liquidity
    - spec / contract / permissions / volume rules
    - sessions / swap / margin

A4. L3 Producer Family
    Owns:
    - filter state
    - eligibility
    - blockers / dependencies
    - gate inputs
    - reason / confidence

A5. L4 Producer Family
    Owns:
    - shortlist status
    - rank
    - priority score
    - hold / stability
    - correlation / exposure

A6. L5 Producer Family
    Owns:
    - deep-analysis entitlement
    - deep-analysis trigger
    - deep-analysis result
    - preservation / continuity
    - current-focus pair truth

----------------------------------------------------------------------
STRAND B — DOSSIER ADAPTERS
----------------------------------------------------------------------

These format producer truth into dossier sections.

Each adapter:
- consumes upstream truth
- performs formatting only
- does not own truth
- does not query random broker data ad hoc unless that adapter is explicitly wrapping the lawful producer contract
- does not decide policy

Adapters should exist for:

B1. Top Header
B2. Operator Snapshot
B3. Publication Health
B4. L0 Identity
B5. L1 Market State
B6. L2 Market Watch
B7. L2 Trading Conditions & Contract
B8. L2 Sessions / Swap / Margin
B9. L3 Candidate Filtering
B10. L4 Shortlist Selection
B11. L5 Deep Selective Analysis
B12. Trader Context
B13. Change Trace

----------------------------------------------------------------------
STRAND C — DOSSIER COMPOSER / PRODUCT MODES
----------------------------------------------------------------------

This assembles the visible dossier.

It decides:
- open-mode vs closed-mode dossier
- section inclusion / exclusion
- section order
- subgroup order
- field label contract
- truthful missingness wording
- shell fallback vs canonical product routing

It does NOT decide:
- market truth
- shortlist truth
- deep-analysis truth
- spec truth
- session truth

----------------------------------------------------------------------
STRAND D — PUBLICATION / RUNTIME ADMISSION
----------------------------------------------------------------------

This owns:
- dossier existence duty
- bootstrap flood coverage
- canonical continuation
- write budgeting
- retries / rescheduling
- selected-symbol dossier/current-focus pair write
- explicit deep-analysis upgrade write
- atomic write / validate / promote

----------------------------------------------------------------------
STRAND E — VALIDATION
----------------------------------------------------------------------

This owns:
- compile truth
- active owner path
- active caller path
- runtime counters
- broker comparison
- mismatch ledger
- regression traps

======================================================================
3. RECOVERY ORDER
======================================================================

The recovery order must be PRODUCER-FIRST.

NOT this:
- make dossier look good
- hope upstream truth catches up later

INSTEAD:
1. recover truth owner
2. recover contract/packet
3. recover dossier adapter
4. recover composer/product mode
5. recover publication admission
6. validate against broker truth

- contradict
- no patch

B. DESIGN ONLY
- define contract
