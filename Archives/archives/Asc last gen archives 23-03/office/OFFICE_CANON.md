# Office Canon

## Purpose

This file is the active office constitution for the repo.
It keeps control compact while reflecting the new dispatcher-first and layer-owned blueprint shape.

## Active office model

The active office layer is:
- one canon
- one task board
- one decision log
- one work log
- one SHA ledger

That is enough.

## Current office target

The current active target is:
- one-dispatcher MT5 runtime
- Layer-1-first startup acquisition
- lane-based scheduler doctrine
- focused-symbol fast HUD paint without broad background churn
- Layer 4 top-5-per-main-bucket ownership
- Layer 5 rank-sensitive promoted-set cadence
- dossiers as consumer surfaces of layer-owned truth blocks

## Update law

Any meaningful repo change should touch office in a bounded way:
- `TASK_BOARD.md` if task state changes
- `DECISIONS.md` if a lock changes
- `WORK_LOG.md` for actual work performed
- `SHA_LEDGER.md` for changed-file checkpoints when needed

## Small-file rule

Do not grow this folder by default.
A new office file is allowed only if it carries a truly distinct long-lived role that does not fit inside the existing canon.

## Current control note

The active repo direction now rejects:
- giant blended heartbeat passes
- warmup exit keyed to non-Layer-1 work
- HUD-driven compute sprawl
- dossier-first truth creation
- later-layer recomputation triggered by page visibility alone
