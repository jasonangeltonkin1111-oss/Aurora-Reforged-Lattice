# Future Blindspot Register

| Blindspot | Why it matters | Preventive rule |
|---|---|---|
| L3 too stale between 20-minute refreshes | Fast markets can shift | L2/L5 active set refreshes stay frequent; manual refresh allowed later if proven needed |
| L4 active set starves non-active symbols | New opportunities may wait for next L3 | L3 universe refresh remains full-universe every 20 min |
| Minor buckets ignored by L5 | Niche opportunities can be missed | minor leaders can be manual deep-review candidates |
| Over-atomic writing slows runtime | Too much flush/write cost | skip unchanged payloads; flush valuable public artifacts only |
| Indicator handles leak | MT5 resources degrade | registry releases handles on deinit |
| Public samples leak account number | repo is public | samples mask account login/name |
| Locked modules become too rigid | valid dependencies can change | lock-break protocol with tests and rollback |
| Codex overbuilds scaffold | architecture infection | scaffold prompts forbid formulas/trading/HUD/direct ASC copy |
| Archive knowledge becomes authority | old debt returns | Archives evidence only; migration register required |
